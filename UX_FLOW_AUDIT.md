> Дата: 2026-05-10. Ветка: `feature/editor-mobile-redesign`.

## Карта флоу

```
Landing (phone input) → Dashboard (index.html)
  ├── Создать → Templates → Editor (mobile-editor.js) → Save → Dashboard
  ├── Гости → guests.html
  ├── Профиль → (logout)
  └── Активировать → paywall.html
```

---

## 🔴 Критично

### 1. Нет реальной аутентификации
**Файл:** `initAuth.js`, API-заголовок `X-User-Phone`

Телефон сохраняется в `localStorage` без OTP/SMS-верификации. Бэкенд принимает `X-User-Phone` на слово. Любой пользователь может ввести чужой номер и получить доступ к его событиям и гостям.

**Исправление:** Добавить OTP-верификацию или JWT-сессию.

---

### 2. XSS через innerHTML без экранирования
**Файл:** `dashboard.js` — `renderEventCard`, `renderEventHub`, `openGuestsSheet`

`event.title`, `event.person1`, `event.person2`, `event.location` вставляются напрямую в `innerHTML`. Если в названии события есть `<img onerror="...">` — выполнится произвольный JS.

```js
// dashboard.js:343 — пример уязвимости
<h3 ...>${event.title || 'Без названия'}</h3>  // ← не экранировано
```

**Исправление:** Экранировать все данные от сервера через `esc()` (функция уже есть в `mobile-editor.js:161`).

## 🟠 Высокий приоритет

### 3. `editor.html` без auth guard
**Файл:** `editor.html` (vs `index.html:11`, `guests.html:11`)

`index.html` и `guests.html` имеют inline-скрипт, который немедленно редиректит неавторизованных. `editor.html` — нет. Пользователь попадает в редактор, пытается сохранить, получает toast "Войдите в аккаунт чтобы сохранить" вместо нормального редиректа.

```html
<!-- Отсутствует в editor.html, но есть в index.html: -->
<script>
  (function () {
    try { if (!localStorage.getItem('tl_phone')) location.replace('/landing.html'); } catch (_) {}
  })();
</script>
```

---

### 4. Нет предупреждения при уходе из редактора с несохранёнными данными
**Файл:** `mobile-editor.js` — флаг `isDirty` отслеживается, но...

Кнопка "Назад" (`<a href="/dashboard.html">`) — это обычная ссылка. При `isDirty = true` пользователь уходит без предупреждения и теряет правки.

**Исправление:**
```js
window.addEventListener('beforeunload', (e) => {
  if (isDirty) e.preventDefault();
});
```
И перехватить клик на кнопке "Назад".

---

### 5. Шаринг DRAFT-события раскрывает preview-ссылку
**Файл:** `dashboard.js:513-515`

В `renderEventHub` WhatsApp/Telegram-кнопки шеаринга используют `eventUrl`, который для DRAFT-события содержит `?preview=TOKEN`. Нажав «Поделиться» → WA, организатор отправляет гостям превью-ссылку с токеном, которая работает до публикации. Скорее всего, непреднамеренное поведение.

## 🟡 Средний приоритет

### 6. Диалог гостей открывается дважды при кеш-хите
**Файл:** `dashboard.js:841-856`

```js
if (cachedEvent && cachedGuests) {
  openGuestsSheet(cachedEvent, cachedGuests);  // 1-й открыт
}

try {
  const [events, guests] = await Promise.all([...]);
  openGuestsSheet(event, guests);  // 2-й открывает поверх
}
```

Это создаёт визуальный артефакт: шит мелькает. Лучше: открыть сразу с кешем, а при получении свежих данных — тихо обновить содержимое внутри уже открытого шита.

---

### 7. `editor.html` «Назад» ведёт через лишний редирект
**Файл:** `editor.html:19`

```html
<a ... href="/dashboard.html">Назад</a>
```

`/dashboard.html` — это просто `location.replace('/')`. Лишний HTTP-запрос и мерцание.

**Исправление:** Заменить на `href="/"`.

---

### 8. Кнопка "К не ответившим" работает, но требует проверки
**Файл:** `dashboard.js:473`, `guests.js:748`

CTA ссылается на `?filter=noReply`. В `guests.js` параметр читается (строка 748), фильтр применяется. Работает, но если пользователь кешировал страницу гостей без этого параметра — отобразится кеш без фильтра, и только после ревалидации фильтр применится. Незначительно, но заметно.

## 🟢 Низкий приоритет / UX

### 9. Демо-данные в редакторе при создании нового события
**Файл:** `mobile-editor.js:30-78`

Редактор всегда инициализируется с «Улансын & Эльнура» вместо пустых полей. Новый пользователь видит чужие имена и думает, что что-то не то. Надо либо очищать поля, либо ставить placeholders.

---

### 10. Текст payment strip не учитывает множественное число
**Файл:** `index.html:1025`

```html
<p>Ваш сайт не активирован</p>
```

Показывается всегда, даже если у пользователя 3 черновика. Можно показать количество.

---

### 11. Мёртвый код — `MANUAL` source в `guestRow`
**Файл:** `dashboard.js:236`

```js
guestSource(g) === 'MANUAL'  // ← это условие никогда не выполнится
```

Функция `guestSource()` возвращает только `PERSONAL_LINK` или `PUBLIC_LINK`. Ветка `MANUAL` недостижима.

---

### 12. Аватар в профиле — последняя цифра номера
**Файл:** `index.html:1141`

```js
const ch = (phone.replace(/\D/g, '').slice(-1) || '?').toUpperCase();
```

Аватар показывает последнюю цифру номера телефона (например, "7"). Лучше показывать первую букву имени, если оно будет добавлено, или просто иконку.
