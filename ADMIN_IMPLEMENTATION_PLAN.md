# ADMIN IMPLEMENTATION PLAN — MVP

## Архитектурные решения

- **Один файл:** `admin.html` — SPA с сайдбаром, все разделы в одном документе.
- **Бэкенд:** `/api/admin/**` — новые контроллеры, проверка `role=SUPERADMIN`.
- **Авторизация:** та же сессионная модель (HttpSession), что и у клиента. Роль SUPERADMIN в БД.
- **Стили:** Tailwind CDN (уже подключён в проекте) + минимальный custom CSS.

## Что делаем (порядок)

### Фаза 1: Бэкенд
1. UserService — роль из БД в UserDetails
2. SecurityConfig — `/api/admin/**` только для SUPERADMIN
3. AdminController — дашборд, пользователи, события, оплаты
4. Liquibase миграция — поля для админки (last_login_at, deleted_at)

### Фаза 2: Фронтенд
5. `admin.html` — логин + SPA с сайдбаром
6. Разделы: Дашборд, Пользователи, События, Оплаты
7. Стили admin (desktop-first)

### Фаза 3: Проверка
8. Компиляция, тестовый запуск

## Что НЕ делаем в MVP
- Impersonation
- CMS лендинга
- Аналитика (кроме простых KPI)
- Audit log (задел в БД, UI потом)
- Промокоды, тарифы, фича-флаги
- Медиа-менеджер
