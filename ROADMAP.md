# ToiLink — Беклог функционала

Оценки: сложность — **Н** (низкая, до 2 дней) / **С** (средняя, 3–7 дней) / **В** (высокая, 1–3 недели).  
Польза — **средняя** / **высокая** / **критическая**.  
Порядок внутри каждого раздела — от приоритетного к второстепенному.

---

## 1. Управление гостями

| # | Функция | Сложность | Польза | Описание |
|---|---------|-----------|--------|----------|
| G-1 | **Сторона гостя** | Н | Высокая | Поле `side` на госте: жених / невеста / общий / другое. Основа для фильтрации, рассадки, статистики баланса. |
| G-2 | **Редактирование гостя** | Н | Высокая | Сейчас есть только удаление. Нужен edit: имя, телефон, сторона, связь, примечание. |
| G-3 | **Связь между гостями** | Н | Высокая | Поля `related_to_id` + `relation_type` (супруг/а, семья, ребёнок, друг, сопровождающий). Метка в списке: «супруга Азамата». |
| G-4 | **RSVP с именами спутников** | С | Высокая | Если groupSize > 1 — гость вводит имена спутников. Каждый создаётся как отдельный Guest с `source=RSVP_COMPANION` и ссылкой на основного. |
| G-5 | **Импорт гостей из Excel/CSV** | С | Высокая | Загрузить список колонками: имя, телефон, сторона, примечание. Скачать шаблон. Без импорта организаторы тратят час на ввод 200 гостей вручную. |
| G-6 | **Экспорт гостей в Excel** | Н | Высокая | Скачать итоговый список: имя, телефон, статус RSVP, стол, сторона. Организаторы передают это в ресторан / тамаде. |
| G-7 | **Теги / группы гостей** | Н | Средняя | Свободные метки: «коллеги», «институт», «родня». Помогают при рассадке и фильтрации. Хранить как `TEXT[]` в PostgreSQL. |
| G-8 | **Определение дублей** | С | Средняя | Сравнивать по номеру телефона при добавлении. Показать предупреждение, если такой телефон уже есть. Не сливать автоматически — только предупреждать. |
| G-9 | **Массовые операции** | С | Средняя | Выбрать несколько гостей → удалить, пересадить за стол, изменить сторону. Нужно когда гостей больше 100. |
| G-10 | **Пищевые предпочтения** | Н | Средняя | Поле на госте: стандартное / вегетарианское / халяль / без глютена. Полезно для передачи в ресторан. |

## 2. RSVP и коммуникация

| # | Функция | Сложность | Польза | Описание |
|---|---------|-----------|--------|----------|
| R-1 | **WhatsApp-рассылка ссылок** | С | Критическая | Кнопка «Отправить в WhatsApp» для каждого гостя или сразу для всех. В КГ WhatsApp = основной канал. Без этого ссылки отправляются вручную. |
| R-2 | **Красивая страница после RSVP** | Н | Высокая | Экран «Ждём вас!» с именем гостя и деталями события. Сейчас возвращается просто JSON. |
| R-3 | **Статус приглашения в списке** | Н | Высокая | Визуально различать: приглашён / подтвердил / отказал / не ответил / возможно. Цветные метки, сейчас частично есть. |
| R-4 | **Кастомный текст приглашения** | Н | Высокая | Организатор пишет текст, который гость получит вместе со ссылкой. Сейчас ссылка идёт без контекста. |
| R-5 | **Таймер RSVP-дедлайна** | Н | Высокая | Показывать на RSVP-форме: «Осталось 5 дней для ответа». Повышает конверсию ответов. |
| R-6 | **Напоминание не ответившим** | С | Высокая | Список гостей без ответа → кнопка «Напомнить» → открывает WhatsApp с предзаполненным текстом. |
| R-7 | **Уведомление организатору** | С | Высокая | Когда гость отвечает — уведомление в браузер (Push API) или Telegram-бот: «Азамат подтвердил участие». |
| R-8 | **QR-код приглашения** | Н | Средняя | Генерировать QR на гостевую страницу. Распечатать и вложить в бумажное приглашение. |
| R-9 | **Календарное приглашение (.ics)** | Н | Средняя | Кнопка «Добавить в календарь» на гостевой странице. Гость сохраняет событие в Google/Apple Calendar. |
| R-10 | **SMS-уведомления** | В | Средняя | Интеграция с Beeline KG / MegaCom. WhatsApp эффективнее и дешевле в реализации. Отложить. |

## 3. Рассадка по столам

| # | Функция | Сложность | Польза | Описание |
|---|---------|-----------|--------|----------|
| S-1 | **CRUD столов** | Н | Критическая | Создавать столы: название («Стол 1», «VIP», «Семья невесты»), вместимость. Фундамент раздела. |
| S-2 | **Kanban-рассадка** | С | Критическая | Колонки: «Без стола» + каждый стол. Перетаскивать гостей. CSS draggable + PATCH запросы. Без Canvas и библиотек. |
| S-3 | **Счётчик мест** | Н | Высокая | На каждом столе: «7 / 10 мест». Красный при переполнении. |
| S-4 | **Фильтр по стороне** | Н | Высокая | В Kanban показывать только гостей одной стороны. Удобно: сначала рассадить сторону жениха, потом невесты. |
| S-5 | **Предупреждение о парах** | Н | Высокая | Если гость перемещается за стол, а его `related_to` уже за другим — тостер «Азамат и его супруга окажутся за разными столами». |
| S-6 | **Печать рассадки** | Н | Высокая | `window.print()` с CSS: каждый стол на отдельном блоке. Без PDF-генерации на сервере. |
| S-7 | **Авторассадка** | В | Средняя | Алгоритм: пары вместе, стороны распределены, вместимость не превышена. Сложно сделать хорошо. Ручной Kanban закрывает 90% потребности. Отложить. |
| S-8 | **Визуальная схема зала** | В | Средняя | Drag-and-drop столов по карте зала с координатами. Красиво, но месяц работы ради фичи для 10% пользователей. Отложить. |

| T-2 | **Блок «Программа дня»** | Н | Высокая | Таймлайн: время → событие. Встречаем гостей, банкет, первый танец. Клиенты просят это чаще всего. |

| T-4 | **Блок «Дресс-код»** | Н | Высокая | Цветовая палитра или текст. Гости часто не знают что надеть — блок снимает неловкость. |

| T-7 | **Countdown-таймер** | Н | Средняя | «До свадьбы осталось X дней» — живая анимация на странице мероприятия. 30 строк JS. |
| T-8 | **Блок «Подарки / wish list»** | Н | Средняя | Ссылки на магазины или реквизиты. Пары стесняются просить — блок снимает неловкость. |
| T-9 | **Шаблоны для других событий** | В | Высокая | Той, юбилей, тұсаукесер, бешик той, корпоратив. Расширяет аудиторию за пределы свадеб. |

## Что не делать

| Функция | Причина |
|---------|---------|
| Fuzzy-dedup гостей по имени | Имена пишутся слишком по-разному. Только по телефону. |
| PDF-генерация рассадки на сервере | `window.print()` + CSS закрывает 95% задачи бесплатно. |
| Авторассадка (алгоритм) | Сложно сделать хорошо. Ручной Kanban быстрее и понятнее. |
| Визуальная схема зала (drag на карте) | Месяц работы ради фичи для 10% пользователей. |
| Прикрепление чеков к расходам | Требует S3 / хранилища. Вернуться после запуска. |
| SMS-рассылка | WhatsApp эффективнее и дешевле в реализации для КГ. |
| Совместный доступ / multi-user роли | Требует системы ролей. Сложно, не нужно сейчас. |
| Маркетплейс шаблонов от дизайнеров | Нужна аудитория для маркетплейса. Сначала аудитория. |

## Порядок реализации (рекомендованный)

```
Волна 1 — Гости (2 недели)
  G-1 сторона гостя
  G-2 редактирование гостя
  G-3 связи между гостями
  R-2 страница после RSVP
  R-3 статусы в списке

Волна 2 — Рассадка (2 недели)
  S-1 CRUD столов
  S-2 Kanban-рассадка
  S-3 счётчик мест
  S-4 фильтр по стороне
  S-5 предупреждение о парах
  S-6 печать

Волна 3 — Смета (1 неделя)
  B-1 CRUD расходов
  B-2 итоги
  B-3 категории

Волна 4 — Коммуникация (1 неделя)
  R-1 WhatsApp-рассылка
  R-4 кастомный текст
  R-6 напоминание не ответившим
  G-5 импорт из Excel
  G-6 экспорт в Excel

Волна 5 — Монетизация (3 недели)
  T-1 шаблон 2
  M-1 тарифные планы
  M-2 платёжный шлюз
  M-3 водяной знак
```
